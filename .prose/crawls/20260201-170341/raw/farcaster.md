## 3. SECURITY - Threats, Vulnerabilities & Scams

### Known Threats

#### Bot Proliferation
- **Severity:** HIGH
- **Status:** Active and growing
- **Description:** After Farcaster opened to masses, bot activity surged significantly. Bots mimic human behavior to spread disinformation, farm engagement, and orchestrate scams/phishing campaigns
- **Impact:** Users report noticeable increase in spam and automated accounts across feeds
- **Source:** [Bots invading Farcaster - DLNews](https://www.dlnews.com/articles/web3/farcaster-users-could-use-frames-and-nfts-to-stop-bots/)

#### Spam & Scam Activity
- **Severity:** MEDIUM-HIGH
- **Status:** Active
- **Vectors:**
  - Token creation scams using Clanker-like tools
  - Phishing via malicious Frames
  - Pump-and-dump schemes via meme coin launches
  - Fake agent impersonation
- **Example:** Multiple tokens launched via Clanker with zero utility or exit scams

#### Frame Abuse
- **Severity:** MEDIUM
- **Status:** Active
- **Description:** Frames were abused for scams, phishing, and engagement farming in 2025
- **Impact:** Power badges and frame verification became contentious trust signals

#### Power Badge Centralization Concerns
- **Severity:** MEDIUM
- **Status:** Ongoing
- **Description:** Power Badge system (intended to surface quality, reduce bot visibility) accused of centralized moderation; badges removed from critical voices
- **Impact:** Community trust erosion; centralization concerns on "decentralized" platform

### Defenses & Mitigation Strategies

#### Protocol-Level Defenses
- **$5 sign-up fee:** Prevents mass spam account creation; creates financial friction
- **Signed payload validation:** Frames require cryptographically signed requests from verified user accounts (similar to CAPTCHA)
- **Key management system:** Smart contracts for identity and key management
- **Bug bounty program:** Immunefi-hosted program for security researchers
- **Source:** [Immunefi - Farcaster Bug Bounties](https://immunefi.com/bug-bounty/farcaster/scope/)

#### Community-Driven Defenses
- **Open-source design:** Users can deploy custom clients/filters to exclude bots
- **Reputation systems:** Community curators can build blocklists and filtered feeds
- **Proof-of-personhood:** Platforms like Worldcoin integration being explored
- **Verification features:** On-chain verification system (Farcaster Verification Protocol) requires minimum wallet holdings to filter malicious actors
- **Source:** [Bots invading Farcaster - DLNews](https://www.dlnews.com/articles/web3/farcaster-users-could-use-frames-and-nfts-to-stop-bots/)

#### Verification & Trust Infrastructure
- **Verified accounts:** Higher visibility in search/feeds; requires identity verification
- **Newton Protocol:** Integrates Neynar data for onchain Farcaster identity guardrails
- **Neynar reputation scoring:** Infrastructure provider offers reputation scoring for trust assessment
- **Gitcoin Passport integration:** Available for proof-of-personhood
- **Source:** [Farcaster Introduces Verification Feature](https://www.ainvest.com/news/farcaster-introduces-verification-feature-enhanced-security-rewards-2505/)
