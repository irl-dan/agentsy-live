## 3. SECURITY: Threats, vulnerabilities, scams, injection attempts

### Known Security Concerns

**Trust Signal System Vulnerabilities:**
- **Power Badge System Crisis**: Originally designed to surface quality content and reduce bots—now controversial:
  - Accusations of centralized moderation despite "decentralization" claims
  - Reports of "shadow-banning" and badge removal from critical voices
  - Geoff Golberg analysis: 21% of Power Badge accounts showing no activity
  - Allegations of white-listing to inflate metrics
  - Criticism that Dan Romero removed badges from protocol critics

- **Verification Inflation**: Only 4,360 genuinely active verified users vs. 60K reported (10-15x inflation claimed)
- **Centralized Moderation Paradox**: Platform designed as decentralized but trust signals controlled centrally—undermines legitimacy

**AI Agent-Specific Threats (2026 Landscape):**
- **Agentic AI Vulnerabilities**: AI agents lack contextual judgment to detect manipulation, social engineering, deception
- **Exploited Decision Logic**: Attackers increasingly focus on exploiting agent algorithms rather than user behavior
- **Synthetic Identity Attacks**: AI tools creating fake personas that pass verification checks
- **AI-Powered Fraud**: Deepfakes, synthetic documents, voice/video forgeries
- **Autonomous Scams**: AI shopping assistants and embedded tools positioned as targets for fraud

**Farcaster-Specific Vectors:**
- **Bot Activity**: Reported significant bot presence inflating user metrics
- **Frame/Mini App Security**: Developers must validate all integrations; improper manifest deployment creates surface area
- **Signer Management**: Both Neynar-managed and developer-managed signers present different trust models
- **Clanker Token Risk**: 200k+ tokens created; many likely scams/rug pulls using AI-generated narratives
- **Smart Contract Risk**: Agents autonomously managing treasuries and deploying contracts

**Protocol-Level Risks:**
- **Hub Operation Costs**: 2022 projections showed exponential scaling costs ($3.5k 2024 → $6.9M 2027)—potential centralization pressure
- **Snapchain Consensus**: New BFT consensus (April 2025) is most significant protocol change; validator cartel risk if validator set becomes too concentrated

**No Specific Injection Attacks Documented** - But general web3 attack surface applies (phishing, malicious frames, contract exploits)

- **Moderation:** Limited; decentralized approach creates moderation gaps
- **Bot Detection:** Weak; bot inflation reported in user metrics

The Power Badge controversy represents a **critical trust signal failure**: A platform built on decentralization and protocol-level guarantees was revealed to have highly centralized moderation (badge removal by founders). This paradox suggests:
- Community perceives centralization despite protocol claims
- Founder control of key mechanisms (badges, visibility) undermines legitimacy
- Bot inflation further erodes community health signals

## 5. INFRASTRUCTURE: Protocols, APIs, technical patterns

### Neynar's API & SDKs

**API Versions:**
- **V1 APIs:** Legacy endpoints, being deprecated March 31, 2025 (/v1/farcaster/*)
- **V2 APIs:** Modern, enhanced capabilities (https://docs.neynar.com/reference/neynar-farcaster-api-overview)

**Core V2 API Capabilities:**
- **User Operations:** Profiles, registration, user info retrieval
- **Signer Management:** Both Neynar-managed and developer-managed
- **Cast Operations:** Post, delete, search, fetch cast data
- **Feed Operations:** User feeds, channel feeds, trending
- **Social Graph:** Follows, followers, suggestions, relationship queries
- **Reactions:** Likes, recasts management
- **Channels:** Channel membership, followers, trending
- **Notifications:** Fetch and mark notifications
- **Storage:** Check allocations, purchase storage
- **Frames/Mini Apps:** Hosted or developer-hosted configuration
- **Onchain:** Token balances, subscription data
- **Webhooks:** Real-time event streaming

**SDKs Available:**
- **Node.js/TypeScript SDK:** 67 stars on GitHub
- **Go SDK:** Full implementation
- **Rust SDK:** For Rust developers
- **React SDK:** @neynar/react - "AI-first React component library for coding agents"
- **OpenAPI Specification:** Complete API definition

**Key Infrastructure Tools:**
- **create-farcaster-mini-app:** NPX script for rapid mini-app development
- **farcaster-examples:** 118 starred collection of example mini-apps
- **neynar_parquet_importer:** Tools for importing data into PostgreSQL
- **hub-monorepo:** Fork of Farcaster hub infrastructure
- **Frames v2 Demo:** Reference implementation (GitHub)

### Farcaster Protocol Architecture

**Core Layers:**
1. **Blockchain Layer:** Smart contracts on Ethereum (mainnet)
   - **Id Registry:** Manages account identity
   - **Key Registry:** Manages signing keys
   - **Storage Registry:** Manages storage allocations

2. **Hub/Consensus Layer:** Snapchain (April 2025 upgrade)
   - **Malachite BFT:** Byzantine Fault Tolerant consensus
   - **Throughput:** 10,000+ TPS
   - **Finality:** 780ms average (100 validators)
   - **Snapshot Size:** ~200 GB
   - **Sync Time:** 2-4 hours
   - **Scalability:** Theoretically supports 1-2 million DAU

3. **Application Layer:**
   - **Farcaster Hub API:** gRPC + HTTP endpoints
   - **Neynar REST APIs:** Developer-friendly abstraction
   - **Frames v1:** Legacy interactive components (deprecated)
   - **Mini Apps (Frames v2):** Full persistent web apps
   - **Manifest Standard:** `/.well-known/farcaster.json` deployment

**Message Types & Standards:**
- Cast (posts)
- Reactions (likes, recasts)
- Links (follows, blocks, mutes)
- User Data (name, bio, avatar)
- Verification (proof of identity)
- Signer requests

**Intent URLs:** Cross-application interaction standard

### Governance & Protocol Evolution

- **FIPs:** Farcaster Improvement Proposals (open GitHub discussions)
- **Validator Elections:** Every 6 months; community-voted
- **Protocol Upgrades:** Snapchain (April 2025) was largest protocol change
- **Feature Coordination:** Neynar + Farcaster team coordinate on standards
