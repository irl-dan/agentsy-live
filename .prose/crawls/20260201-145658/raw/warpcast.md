## 5. INFRASTRUCTURE: Protocols, APIs, Technical Patterns

### Protocol Architecture

**Core Protocol:**
- **Type:** Decentralized social protocol (Farcaster)
- **Primary Client:** Warpcast (web3 company's application)
- **Network:** Hubble network of nodes
- **Specification:** Available on GitHub: https://github.com/farcasterxyz/protocol/blob/main/docs/SPECIFICATION.md

**Message System:**
- Cryptographically signed binary objects (proto3 protobufs)
- Signing schemes: ED25519 or EIP712
- Each message represents delta-operation on the network

**On-Chain Infrastructure:**
- **Id Registry Contract:** Tracks account IDs (fids)
- **Key Registry Contract:** Manages keys for fids
- **Storage Registry Contract:** Allocates storage to fids
- All contracts are open, auditable, on-chain

**API Layer:**
- **Client APIs:** Full documentation at https://docs.farcaster.xyz/reference/farcaster/api
